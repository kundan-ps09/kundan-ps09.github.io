History of Communiqué

This product was originaly started by a Swiss company Day Software in early 2000 with the name "Communique". CQ is the short form of "Communique".

Comunique (CQ) 1.0

  • Started by a Swiss company Day Software
  • CGI Scripts for Netscape Enterprise Server

Comunique (CQ) 2.0

  • Standalone application (single binary)
  • C Programming Language
  • HTTP Server embedded
  • File system based page (content) storage

Comunique (CQ) 3.0

  • Communiqué 2 rewritten as a Web Application in Java
  • HTTP Server separated out (birth of CQSE)
  • Content Bus storage (connectors for various file formats, databases, etc)
  • File system based page (content) storage
  • Released in 2002

Day Comunique (CQ) 4.0

  • Content Bus storage over JCR (CRX 1.0)
  • Two Web Applications: CRX and CQ
  • Released in 2005

Day Comunique (CQ) 5.0

  • Complete Rewrite of (almost) everything
  • Released in 2008

Day Comunique (CQ) 5.2

  • Release with lot of bug fixes
  • Released in 2009

Day Comunique (CQ) 5.3

  • Release with lot of bug fixes
  • Released in 2010

Adobe CQ 5.4

  • Adobe Systems renamed Day CQ5 as Adobe CQ5 after acquiring Day Software in 2010
  • Released in 2011

Adobe CQ 5.5

  • New architecture- OSGI based architecture
  • Released in 2012

Adobe Experience Manager (AEM) 5.6

  • Adobe Release CQ5 with new Name
  • Introduction to Touch UI
  • Released in 2013

Adobe Experience Manager (AEM) 6.0

  • Replaced CQSE with Jetty
  • New templating mark-up language- Sightly
  • Jackrabbit Oak (aka Jackrabbit 3)
  • Touch UI became functional
  • New implementiation for Social Component Framework (using handlebar JS templating Language )
  • Released in 2014

Adobe Experience Manager (AEM) 6.1

  • Improved Touch UI Authoring interface(usable)
  • Released in 2015

Adobe Experience Manager (AEM) 6.2

  • Release with bug fixes 6.1
  • Released in 2016
